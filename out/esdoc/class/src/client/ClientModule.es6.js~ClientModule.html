    ClientModule

    Extends:

    EventEmitter → src/client/ClientModule.es6.js~Promised → ClientModule

    Direct Subclass:

    ClientCalibration, src/client/ClientCheckin.es6.js~ClientCheckin, ClientControl, ClientDialog, ClientLoader, ClientLocator, ClientOrientation, ClientPerformance, ClientPlacer, ClientPlatform, ClientSelector, ClientSetup, src/client/ClientSpace.es6.js~ClientSpace, src/client/ClientSurvey.es6.js~ClientSurvey, ClientSync, Filelist

    The ClientModule base class is used to create a Soundworks module on the client side. Each module should have a <span><a href="class/src/client/ClientModule.es6.js~ClientModule.html#instance-method-start">ClientModule#start</a></span> and a <span><a href="class/src/client/ClientModule.es6.js~ClientModule.html#instance-method-done">ClientModule#done</a></span> method. The <span><a href="class/src/client/ClientModule.es6.js~ClientModule.html#instance-method-done">ClientModule#done</a></span> method must be called when the module can hand over the control to the subsequent modules (i.e. when the module has done its duty, or when it may run in the background for the rest of the scenario after it finished its initialization process). The base class optionally creates a view — a fullscreen div accessible through the this.view attribute — that is added to the DOM when the module is started and removed when the module calls its <span><a href="class/src/client/ClientModule.es6.js~ClientModule.html#instance-method-done">ClientModule#done</a></span> method. (Specifically, the view element is added to the #container DOM element.)

    Example:

    class MyModule extends ClientModule {
  constructor(options = {}) {
    // Here, MyModule would always have a view,
    // with the id and class 'my-module-name',
    // and possibly the background color
    // defined by the argument 'options'.
    super('my-module-name', true, options.color || 'alizarin');

    ... // anything the constructor needs
  }

  start() {
    super.start();

    ... // what the module has to do
        // (communication with the server, etc.)

    this.done(); // call this method when the module can
                 // hand over the control to a subsequent module
  }
}

    public name: String source

    Name of the module.

    public view: DOMElement source

    View of the module. The view is a DOM element (a full screen div) in which the content of the module is displayed. This element is a child of the main container div#container, which is the only child of the body element. A module may or may not have a view, as indicated by the argument hasView:Boolean of the <span><a href="class/src/client/ClientModule.es6.js~ClientModule.html#instance-constructor-constructor">ClientModule#constructor</a></span>. When that is the case, the view is created and added to the DOM when the <span><a href="class/src/client/ClientModule.es6.js~ClientModule.html#instance-method-start">ClientModule#start</a></span> method is called, and is removed from the DOM when the <span><a href="class/src/client/ClientModule.es6.js~ClientModule.html#instance-method-done">ClientModule#done</a></span> method is called.

    Should be called when the module can hand over the control to a subsequent module (for instance at the end of the start method you write). If the module has a view, the done method removes it from the DOM.

    public abstract start() source

    Handles the logic of the module on the client side. For instance, it takes care of the communication with the module on the server side by sending WebSocket messages and setting up WebSocket message listeners. Additionally, if the module has a view, the start method creates the corresponding HTML element and appends it to the DOM’s main container element (div#container).

    Note: the method is called automatically when necessary, you should not call it manually.
