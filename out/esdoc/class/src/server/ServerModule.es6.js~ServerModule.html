    ServerModule

    Extends:

    EventEmitter → ServerModule

    Direct Subclass:

    ServerCalibration, ServerControl, src/server/ServerPerformance.es6.js~ServerPerformance, ServerSync

    The ServerModule base class is used to create a Soundworks module on the server side. Each module should have a <span><a href="class/src/server/ServerModule.es6.js~ServerModule.html#instance-method-connect">ServerModule#connect</a></span> and a <span><a href="class/src/server/ServerModule.es6.js~ServerModule.html#instance-method-disconnect">ServerModule#disconnect</a></span> method. Any module mapped to the type of client clientType (thanks to the <span>server#map</span> method) would call its <span><a href="class/src/server/ServerModule.es6.js~ServerModule.html#instance-method-connect">ServerModule#connect</a></span> method when such a client connects to the server, and its <span><a href="class/src/server/ServerModule.es6.js~ServerModule.html#instance-method-disconnect">ServerModule#disconnect</a></span> method when such a client disconnects from the server.

    Example:

    class MyModule extends serverSide.Module {
  constructor('my-module-name') {
    ... // anything the constructor needs
  }

  connect(client) {
    ... // what the module has to do when a client connects to the server
  }

  disconnect(client) {
    ... // what the module has to do when a client disconnects from the server
  }
}

    Public Methods

    public connect(client: ServerClient) source

    Called when the client connects to the server. This method should handle the logic of the module on the server side. For instance, it can take care of the communication with the client side module by setting up WebSocket message listeners and sending WebSocket messages, or it can add the client to a list to keep track of all the connected clients.

    Params:

    NameTypeAttributeDescription
    client ServerClient

    The connected client.

    public disconnect(client: ServerClient) source

    Called when the client client disconnects from the server. This method should handle the logic when that happens. For instance, it can remove the socket message listeners, or remove the client from the list that keeps track of the connected clients.

    Params:

    NameTypeAttributeDescription
    client ServerClient

    The disconnected client.
