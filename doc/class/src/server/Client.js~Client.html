    Client

    [server] Client who connects to the server.

    Each time a client of type 'clientType' connects to the server, Soundworks creates a new instance of Client. An instance of the class is passed to the connect and disconnect methods of all the server side modules that are mapped to the 'clientType' clients (see server#map), as well as to the enter and exit methods of any src/server/Performance.js~Performance class mapped to that same client type.

    The class is also used to communicate with the client via WebSockets.

    Example:

    class MyPerformance extends Performance {
  // ...

  enter(client) {
    const msg = "Welcome to the performance!";
    client.send('init', msg);
  }

  // ...
}

    public modules: Object source

    Used by any src/server/Module.js~Module to associate data to a particular client.

    All the data associated with a module whose name is 'moduleName' is accessible through the key moduleName. For instance, the src/server/Checkin.js~Checkin module keeps track of client's checkin index and label in this.modules.checkin.index and this.modules.checkin.label. Similarly, a src/server/Performance.js~Performance module whose name is 'myPerformance' could report the client's status in this.modules.myPerformance.status.
