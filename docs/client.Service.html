ready()

Source:
  • client/Service.js, line 130

Method to call in the service lifecycle when it should be considered as ready and thus allow the initialization process to continue or the application to start safely (cf. client.Experience#start)

Example
class MyDelayService extends soundworks.Service {
  // ...
  async start() {
    this.state = await this.client.stateManager.attach(`s:${this.name}`);
    this.started();
    // do [async] stuff
    setTimeout(() => this.ready(), 3000);
  }
}

start()

Interface method to override when implementing child class.

The child class MUST call client.Service#started and client.Service#ready

Example
class MyDelayService extends soundworks.Service {
  // ...
  async start() {
    this.state = await this.client.stateManager.attach(`s:${this.name}`);
    this.started();
    // do [async] stuff
    setTimeout(() => this.ready(), 3000);
  }
}

started()

Source:
  • client/Service.js, line 97

Method to call when the plugin is effectively started, as it may have to do some asynchonous job at start (e.g. creating a shared state).

Must be called between before client.Service#ready.

Example
class MyDelayService extends soundworks.Service {
  // start() is executed when the `start` signal pass to `true`
  async start() {
    this.state = await this.client.stateManager.attach(`s:${this.name}`);
    this.started();
    // do [async] stuff
    setTimeout(() => this.ready(), 3000);
  }
}
