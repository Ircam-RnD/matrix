ready()

Source:
  • client/AbstractPlugin.js, line 141

Method to call in the plugin lifecycle when it should be considered as ready and thus allow the initialization process to continue or the application to start safely (cf. client.AbstractExperience#start)

Example
class MyDelayPlugin extends AbstractPlugin {
  // ...
  async start() {
    this.state = await this.client.stateManager.attach(`s:${this.name}`);
    this.started();
    // do [async] stuff
    setTimeout(() => this.ready(), 3000);
  }
}

  }

started()

Source:
  • client/AbstractPlugin.js, line 108

Method to call when the plugin is effectively started, as it may have to do some asynchonous job at start (e.g. creating a shared state).

Must be called between before client.AbstractPlugin#ready.

Example
class MyDelayPlugin extends AbstractPlugin {
  // start() is executed when the `start` signal pass to `true`
  async start() {
    this.state = await this.client.stateManager.attach(`s:${this.name}`);
    this.started();
    // do [async] stuff
    setTimeout(() => this.ready(), 3000);
  }
}
